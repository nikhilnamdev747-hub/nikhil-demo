Git And GitHub

What is git?

 Git is a version control system that acts like a "save game"
or "time machine" for your computer files, especially code.
It allows you to track every small change you make,
so you can always go back to an earlier version if you make a mistake.

What is git hub?

 GitHub is a web-based platform that acts as a cloud storage and social network
for programmers. It is the most popular place for developers to store their code,
track changes, and work together on projects from anywhere in the world.

Setup

Configuring user information used across all local repositories

git config --global user.name “[firstname lastname]”
set a name that is identifiable for credit when review version history

git config --global user.email “[valid-email]”
set an email address that will be associated with each history marker

SETUP & INIT

Configuring user information, initializing and cloning repositories

1st- git init
initialize an existing directory as a Git repository

2nd- git add .
add a file as it looks now to your next commit (stage)

3rd- git commit -m “firstcommit”
commit your staged content as a new commit snapshot

4th- Git remote add origin (repo link)
add a git URL as an alias

5th- git remote -v
To verify remote

6th- git branch
To check the branch

7th- git branch -M main
To rename the branch

8th- git push -u main
Transmit local branch commits to the remote repository branch

Extra for mac

1si- git fetch origin tmp
retrieves the latest history and data from the main branch

2nd- git rebase tmp
It is used to reapply the commits of your current branch on top of the latest commit of a branch named tmp.

3rd- git push origin head main
It is used to push your current local commits directly to the main branch on the remote repository named origin.
